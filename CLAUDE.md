## Architecture

**Single-page app.** `app/page.tsx` is the only route (`"use client"`). `app/layout.tsx` is a Server Component that injects two inline blocking scripts before React hydrates (see below).

### Store (`store/`)

One Zustand store (`useWeeklyGridStore`) composed of three slices via `persist` middleware:

- `activities-slice` — `ActivityRow[]` CRUD + drag reorder via `@dnd-kit/sortable`'s `arrayMove`
- `locale-slice` — `"en" | "es"` (i18n toggle)
- `theme-slice` — `ThemeId` + `AppearanceMode`

Persisted to `localStorage` under key `"weekly-commit"`. Only rows with a non-empty trimmed name are written to storage (`persistedActivitiesOnly`). Draft rows (name `""`) are in-memory only.

`store/merge-persisted-state.ts` handles safe rehydration — validates shapes before merging, never blindly spreads persisted state.

### Theming & flash prevention

Themes (`paper-planner`, `graphite`, `botanical-flow`) are CSS-only: each `[data-theme="…"]` block in `app/globals.css` defines CSS custom properties (`--theme-page`, `--theme-ink`, `--wc-font-title`, etc.). Light/dark is controlled by `data-appearance="dark"` on `<html>`.

To avoid flash-of-wrong-theme, `app/layout.tsx` injects **two inline blocking scripts** generated by `lib/themes.ts` (`getThemeBootstrapScript`) and `lib/i18n.ts` (`getLocaleBootstrapScript`). These read `localStorage` before React mounts and set `data-theme`, `data-appearance`, and `lang` directly on `<html>`. Google Fonts are also loaded here.

`ThemeAppearanceRoot` (`components/theme-appearance-root.tsx`) re-applies attributes after hydration via `useLayoutEffect`, because React can strip unknown `data-*` on `<html>` during reconciliation.

### i18n

Two locales (`en`, `es`). All UI strings go through `lib/messages.ts` → `getMessages(locale)`. No routing-based i18n — locale is just a store value.

### `ActivityRow` lifecycle

A row exists in one of two states:
- **Draft**: `name === ""` — no checkboxes, no duration, not persisted
- **Complete**: `hasActivityName && durationMinutes != null` — checkboxes enabled, included in time totals

Only one draft row is allowed at a time (`addActivity` guards this).

### Testing

Vitest + Testing Library + jsdom. Setup in `vitest.setup.ts`. Tests live alongside their source files (`.test.ts` / `.test.tsx`). SSR edge cases (no `window`) are covered in lib tests; component tests use jsdom.
